## 2. 云开发环境配置

本项目使用 **微信云开发**（CloudBase），无需单独购买服务器。

### 2.1 开通云开发

1. 在微信开发者工具中，点击左上角 **云开发** 按钮
2. 首次使用会提示开通，点击 **开通** 即可
3. 创建新环境，环境名称建议：`ncard-prod`
4. 环境 ID 会自动生成，类似 `ncard-prod-xxx`

### 2.2 环境配置

在 `miniprogram/app.js` 中，云开发初始化代码已配置：

```javascript
wx.cloud.init({
  traceUser: true,
  env: wx.cloud.DYNAMIC_CURRENT_ENV  // 自动使用当前环境
})
```

> 💡 `DYNAMIC_CURRENT_ENV` 表示跟随开发者工具中选中的环境。正式发布后，建议改为固定环境 ID 以提升稳定性。

### 2.3 云存储配置

1. 进入云开发控制台 → **存储**
2. 创建以下存储目录：
   - `avatars/` — 用户头像（由代码自动创建，无需手动操作）
   - `attachments/` — 名片附件文件
   - `qrcodes/` — 小程序码

3. 设置存储权限：推荐「仅创建者可读写」
   > 头像分享通过云函数 `resolveCloudUrls` 代理读取，无需开放公开读权限

## 3. 云函数部署

### 3.1 云函数列表

| 云函数 | 用途 | 需要部署 |
|--------|------|----------|
| `getOpenId` | 获取用户 OpenID | ✅ |
| `getQrCode` | 生成小程序码 | ✅ |
| `initVisits` | 访客记录读写 + 身份识别 | ✅ |
| `deleteCard` | 级联删除名片（数据库 + 存储） | ✅ |
| `resolveCloudUrls` | 安全代理：cloud:// → HTTPS URL | ✅ |

### 3.2 部署方式

**方式一：逐个上传（推荐）**

在微信开发者工具中：
1. 展开 `cloudfunctions` 目录
2. 右键点击云函数文件夹（如 `getOpenId`）
3. 选择 **上传并部署：云端安装依赖**

**方式二：批量上传**

```bash
# 在项目根目录执行（需要安装 Node.js）
bash uploadCloudFunction.sh
```

### 3.3 部署后验证

进入云开发控制台 → **云函数**，确认每个函数的状态为 **部署成功**。

点击函数名称可查看调用日志，进行功能测试：

```javascript
// 在云函数测试面板中测试 parseCard
// 输入测试参数：
{ "fileID": "cloud://xxx.png" }
```

## 4. 数据库集合配置

### 4.1 需要创建的集合

| 集合名称 | 用途 | 权限 |
|----------|------|------|
| `cards` | 名片信息 | 仅创建者可读写 |
| `scans` | OCR 识别记录 | 仅创建者可读写 |
| `visitors` | 访客记录 | 仅创建者可读写 |
| `users` | 用户信息 | 仅创建者可读写 |

### 4.2 创建步骤

1. 进入云开发控制台 → **数据库**
2. 点击 **添加集合**
3. 输入集合名称，点击确定

### 4.3 集合索引优化

为提高查询性能，建议在云开发控制台为以下字段添加索引：

- `cards` 集合：`_openid`（升序）、`createTime`（降序）
- `scans` 集合：`_openid`（升序）、`createTime`（降序）
- `visitors` 集合：`cardId`（升序）、`visitTime`（降序）

## 5. 域名白名单

由于本项目使用云开发且无外部 API 调用，**通常无需额外配置域名白名单**。

### 5.1 云开发域名（自动配置）

- `https://api.weixin.qq.com` — 微信 API
- `https://tcb-api.tencentcloudapi.com` — 云开发 API
- 云存储域名（自动生成）

### 5.2 如需添加外部 API

登录 [微信公众平台](https://mp.weixin.qq.com) → 开发 → 开发管理 → 开发设置 → 服务器域名：

| 类型 | 配置 |
|------|------|
| request 合法域名 | `https://your-api.com` |
| socket 合法域名 | `wss://your-ws.com` |
| uploadFile 合法域名 | `https://your-upload.com` |
| downloadFile 合法域名 | `https://your-cdn.com` |

> ⚠️ 域名必须为 HTTPS，且已备案。每月最多修改 5 次。

## 6. 隐私协议与权限

### 6.1 用户隐私保护指引

**必须操作：** 登录 [微信公众平台](https://mp.weixin.qq.com) → 设置 → 基本设置 → 服务内容声明 → 用户隐私保护指引。

填写以下信息：
- 收集的用户信息类型：
  - [x] 手机号（名片填写）
  - [x] 头像和昵称（用户信息）
  - [x] 相册（名片图片上传/扫描）
  - [x] 位置信息（如需使用）
- 信息使用目的：用于名片创建、识别和管理
- 第三方 SDK：无

### 6.2 权限声明

在 `miniprogram/app.json` 中添加所需权限：

```json
{
  "permission": {
    "scope.userLocation": {
      "desc": "用于在名片中显示您的位置信息"
    },
    "scope.writePhotosAlbum": {
      "desc": "用于保存名片二维码到相册"
    }
  }
}
```

> ⚠️ 权限描述必须具体、清晰，否则审核可能被拒。

### 6.3 隐私弹窗要求

从 2023年9月15日起，微信要求小程序在调用隐私相关 API 前，必须展示隐私弹窗。

本项目已使用 `wx.requirePrivacyAuthorize` 相关基础库能力（3.0.0+），确保基础库版本 ≥ 3.0.0。

## 8. 上传与版本管理

### 8.1 版本流转

```
开发版 ──→ 体验版 ──→ 审核版 ──→ 线上版
  │          │          │
  └─ 本地    └─ 扫码预览  └─ 提交审核
```

### 8.2 上传代码

1. 在微信开发者工具中，确认所有代码已保存
2. 点击右上角 **上传** 按钮
3. 填写版本号和备注：
   - 版本号格式：`v1.0.0`
   - 备注：清晰描述此版本的改动

### 8.3 版本号规范

采用语义化版本（SemVer）：

| 阶段 | 版本号示例 |
|------|-----------|
| 开发中 | `v0.1.0` ~ `v0.9.x` |
| 首次发布 | `v1.0.0` |
| Bug 修复 | `v1.0.1` |
| 新增功能 | `v1.1.0` |
| 重大更新 | `v2.0.0` |

### 8.4 体验版测试

1. 登录 [微信公众平台](https://mp.weixin.qq.com) → 管理 → 版本管理
2. 在「开发版本」中，选择要测试的版本 → 点击 **选为体验版**
3. 在「成员管理」中添加体验成员（最多 15 人）
4. 体验成员通过小程序码或链接扫码体验

## 9. 提交审核

### 9.1 提审步骤

1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 进入 **管理 → 版本管理**
3. 在「开发版本」中找到目标版本
4. 点击 **提交审核**
5. 填写审核信息：

| 字段 | 填写内容 |
|------|----------|
| 配置功能页面 | 选择至少一个页面（如 `pages/index/index`） |
| 标题 | 科博名片 |
| 所在服务类目 | 工具 > 信息查询 |
| 标签 | 名片, 扫描, OCR |

6. 提交

### 9.2 审核时间

- 一般：**1-7 个工作日**
- 首次提交或涉及敏感类目：可能更长
- 加急审核：每个自然年有 3 次加急机会

### 9.3 审核通过后

1. 审核通过后，状态变为 **待发布**
2. 点击 **发布** 按钮
3. 小程序即正式上线，用户可搜索和使用

> 💡 建议首次发布使用 **灰度发布**，先开放 5%-10% 用户，观察无问题后再全量发布。

## 11. 常见审核拒绝原因

| 问题 | 解决方案 |
|------|----------|
| **服务类目不匹配** | 选择正确的类目，如「工具 > 信息查询」 |
| **功能不完整** | 确保所有页面有实际内容，无空白页或「功能开发中」 |
| **测试数据未清除** | 删除所有 mock 数据、测试账号、占位文字 |
| **缺少隐私政策** | 在后台设置用户隐私保护指引 |
| **诱导分享** | 移除「分享后解锁」「转发得奖励」等文案 |
| **要求强制登录** | 用户在不登录的情况下也能浏览基本功能 |
| **涉及UGC无审核** | 名片文字内容需要过滤敏感词 |
| **虚拟支付问题** | iOS 下不允许虚拟支付，需判断平台处理 |
| **类目资质不全** | 选择不需要特殊资质的类目 |
| **小程序名称侵权** | 确保名称不包含品牌词或他人商标 |

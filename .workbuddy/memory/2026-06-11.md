## 13:39 跨设备名片分享头像不可见修复

### 根因
1. **云存储默认权限**：`wx.cloud.uploadFile()` 上传的文件默认仅上传者可读，其他用户通过 `<image src="cloud://...">` 加载时权限拒绝
2. **缺少加载失败降级**：`<image>` 无 `binderror` 兜底，失败后头像区空白

### 代码修复

| 文件 | 修改内容 |
|------|---------|
| `app.js` | 新增 `resolveCloudFileIDs(fileIDs)` 方法，批量调用 `wx.cloud.getTempFileURL()` 将 `cloud://` ID 转为临时 HTTPS URL |
| `pages/preview/index.js` | `loadCard()` 加载后调用 `_resolveCardAvatar()` 转换头像 URL；新增 `onAvatarError()` 降级为默认头像 |
| `pages/preview/index.wxml` | 头像 `<image>` 添加 `binderror="onAvatarError"` |
| `pages/list/index.js` | `_fetchCardsByIds()` 获取名片后调用 `resolveCloudFileIDs` 批量转换；新增 `onAvatarError()` |
| `pages/list/index.wxml` | 头像 `<image>` 添加 `binderror="onAvatarError" data-index="{{index}}"` |
| `pages/index/index.js` | 新增 `onAvatarError()` 降级处理（首页仅显示自有名片，影响较小但防御性添加） |
| `pages/index/index.wxml` | 头像 `<image>` 添加 `binderror="onAvatarError" data-index="{{index}}"` |

### ⚠️ 必须操作的云开发控制台设置

**云开发 → 存储 → 权限设置 → 将存储桶权限改为：**
- ✅ 所有用户可读，仅创建者及管理员可写

> 这是修复的**关键步骤**。没有此设置，`getTempFileURL()` 对其他用户也会失败。代码中的解析逻辑是防御性措施，与权限设置配合才能完全解决跨设备头像不可见问题。

## 14:37 匿名访客数据追踪与归因分析

### 问题核心
未创建名片的访客查看他人名片时，访客列表显示「微信用户」且无头像、无聚合，访客数据无商业价值。

### 关键发现

**🔴 P0：recordVisit 完全失效**
- `preview/index.js:37`：`res.result?.openid` 解析路径错误
- `getOpenId` 云函数返回 `{ success: true, data: { openid: "..." } }`
- 正确路径应为 `res.result?.data?.openid`
- `visitorOpenId` 始终为空字符串 → 云函数校验失败直接返回
- **结论：访客追踪功能自上线以来从未工作过**

**现有匿名槽位**
visits 集合代码中预留了 `visitorName`/`visitorAvatar`/`visitorPhone`/`visitorPosition`/`visitorCompany` 五个扩展字段，但从未写入，全部走 `|| '微信用户'` fallback。

**来源归因缺失**
`source` 字段硬编码为 `'direct'`，未捕获 `wx.getLaunchOptionsSync()` 的 scene 信息（群聊/单聊/朋友圈/扫码等）。

### 方案设计（三阶段）

**P0 Fix**：1 行代码修复 openid 解析 → 让访客追踪开始工作

**Phase 1：身份识别**
- 三级访客体系：L3 卡片用户（查 cards 集合，显示真名/头像）→ L2 已授权用户（查 visitor_profiles 集合，显示微信昵称/头像）→ L1 纯匿名（显示 `访客 #A3F2`，openid hash 前缀）
- 新建 `visitor_profiles` 集合存储已授权用户的微信昵称/头像
- 扩展现有 visits 的 visitorName/visitorAvatar 等字段在云函数端自动 enrichment
- 前端授权引导采用非阻断式底部通知条

**Phase 2：来源归因**
- recordVisit 增加 enterOptions（scene + shareTicket）
- 云函数端 scene → 中文标签映射（群聊分享/单聊分享/朋友圈/扫码等）

**Phase 3：访客聚合画像**
- 按 visitorOpenId 聚合多次访问记录
- 展示总访问次数/首次末次时间/看过哪些名片/来源渠道

### 隐私合规
- visitor_profiles 仅创建者可读写
- 名片主人通过云函数 enrichment 获得昵称（副本），不直接访问 visitor_profiles
- 隐私政策需补充访客信息收集声明
- 后续需提供用户删除自己访客痕迹的入口
